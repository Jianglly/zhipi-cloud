# 智批云 - 团队协作开发指南

---

## 一、快速开始（新成员 5 分钟上手）

### 前置条件
| 工具 | 版本要求 | 安装方式 |
|------|---------|---------|
| **Python** | 3.10+ | [python.org](https://www.python.org/downloads/) |
| **Node.js** | 18+ | [nodejs.org](https://nodejs.org/) |
| **MySQL** | 8.0+ | [mysql.com](https://dev.mysql.com/downloads/mysql/) |
| **Git** | 2.40+ | [git-scm.com](https://git-scm.com/) |

### 1. 克隆项目
```bash
git clone <GitHub仓库地址> zhipi-cloud
cd zhipi-cloud
```

### 2. 后端初始化
```bash
# 进入后端目录
cd zhipi-cloud/zhipi-backend

# 创建虚拟环境
python -m venv .venv

# 激活虚拟环境 (Windows)
.venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

# 复制环境配置（修改数据库密码等）
copy .env.example .env

# 编辑 .env，填写你的 MySQL 密码
notepad .env
```

### 3. MySQL 数据库初始化
```bash
# 在项目根目录执行
python 初始化数据库.py
```
按提示输入 MySQL 密码即可自动建库建表。

### 4. 前端初始化
```bash
cd zhipi-cloud/zhipi-frontend
npm install
```

### 5. 启动开发服务器
双击 `启动智批云.bat`，选择 **[1] 本地开发模式**。

## 二、Git 协作流程

### 分支策略
```
main           ← 稳定版本，只合并，不直接提交
├── develop    ← 开发主分支
│   ├── feature/xxx      ← 功能分支
│   ├── fix/xxx          ← 修复分支
│   └── ...
```

### 日常开发流程（必读！）

```bash
# 1. 每天早上先拉最新代码
git checkout develop
git pull origin develop

# 2. 创建自己的功能分支
git checkout -b feature/你的功能名

# 3. 写代码...写代码...

# 4. 提交代码
git add .
git commit -m "feat: 完成了xxx功能"

# 5. 下班前推送分支
git push origin feature/你的功能名

# 6. 去 GitHub 创建 Pull Request
# 让另一位队友 Review 后合并到 develop
```

### 提交信息规范
```
feat:    新功能       feat: 添加试卷导出功能
fix:     Bug修复      fix: 修复成绩列表分页错误
docs:    文档更新     docs: 更新API文档
style:   样式调整     style: 统一按钮配色
refactor: 代码重构    refactor: 提取公共验证逻辑
test:    测试相关     test: 添加OCR解析单元测试
```

### 冲突解决原则
- **发生冲突时**：不要自己硬来，拉上写冲突代码的队友一起看
- **合并前**：确保本地跑起来没问题再合并
- **谁的功能谁负责**：你的分支你维护，出问题你跟进

## 三、任务分工建议

根据当前项目模块，三人可以这样分：

| 模块 | 内容 | 难度 |
|------|------|------|
| **后端 API** | FastAPI 接口、数据库操作、认证 | ⭐⭐⭐ |
| **前端页面** | Vue3 组件、页面交互、图表 | ⭐⭐⭐ |
| **OCR / AI 批阅** | 百度 OCR 接入、AI 阅卷逻辑、图像处理 | ⭐⭐⭐⭐ |

### 建议分工方式

**队员 A（后端核心）**：
- 数据库设计与优化
- 新增 API 接口
- 数据统计与分析逻辑

**队员 B（前端核心）**：
- 页面布局与交互
- 图表可视化
- 用户体验优化

**队员 C（AI/算法）**：
- OCR 识别优化
- AI 自动批阅逻辑
- 图像预处理

> 提示：使用 GitHub Issues 来分配任务，每完成一个就 close 一个，进度一目了然。

## 四、开发环境说明

### 配置文件的处理

| 文件 | 是否提交 Git | 说明 |
|------|:----:|------|
| `.env` | ❌ | 包含数据库密码等敏感信息，已在 `.gitignore` |
| `.env.example` | ✅ | 配置模板，不含真实密码 |
| `.env.production` | ✅ | 生产环境构建模板 |
| `frontend_dist/` | ❌ | 前端构建产物，本地生成 |

### 路径说明

所有 Python 脚本已支持**动态路径**：
- `launcher.py` 使用 `os.path.dirname(__file__)` 自动定位项目根目录
- `初始化数据库.py` 同理
- 队友 clone 到任意目录都能正常启动

### 数据库方案

**每个人的本地 MySQL 独立运行**，数据库名统一为 `zhipi_cloud`。

数据库被 `.gitignore` 忽略，不会互相覆盖。
需要共享测试数据时，修改 `SQL/02_seed_data.sql` 提交即可。

## 五、踩坑记录 & 常见问题

### launcher.py 启动失败
- 检查 MySQL 服务是否启动（Win+R → `services.msc` → MySQL80）
- 检查 `.env` 文件是否存在（`backend/.env`）
- 检查 `.venv` 是否已创建并安装依赖

### npm install 速度慢
```bash
npm config set registry https://registry.npmmirror.com
```

### pip install 速度慢
```bash
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

### 端口被占用
```bash
# 查看端口占用
netstat -ano | findstr :8000
# 结束进程 (把 <PID> 替换为实际进程号)
taskkill /PID <PID> /F
```

## 六、开发约定

1. **代码注释用中文**，变量命名用英文
2. **Vue 组件**：一个 `.vue` 文件不超过 500 行
3. **API 路由**：统一 `RESTful` 风格，`/api/资源名/动作`
4. **测试先行**：改完代码先自己跑一遍再 push
5. **Pull Request**：至少一人 Review 后才能合并
