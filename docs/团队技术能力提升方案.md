### 1.3 数据库会话管理修复 (`config/database.py`)

```python
# Before: 仅 close()，异常时不回滚
def get_db():
    db = SessionLocal()
    try: yield db
    finally: db.close()

# After: 自动 commit/rollback + close
def get_db():
    db = SessionLocal()
    try:
        yield db
        db.commit()         # 正常完成 → 提交
    except Exception:
        db.rollback()       # 异常 → 回滚
        raise
    finally:
        db.close()          # 无论如何 → 关闭
```

**控制器变更：** 所有手动 `db.commit()` 已替换为 `db.flush()`（由 `get_db()` 统一管理事务）。

## 三、技术规范指南

### 3.1 后端开发规范

#### 分层架构（必须遵守）

```
Controller (路由层)
  ↓ 调用
Service (业务逻辑层)      ← 所有业务规则在此
  ↓ 调用
Repository/Model (数据层) ← 数据库操作在此
```

**铁律：**
1. **Controller 不包含业务逻辑** — 只做请求解析、参数验证、调用 Service、格式化响应
2. **Service 不导入 HTTP 类型** — 不依赖 `Request`/`Response`/`HTTPException`
3. **所有异常使用 `AppException` 子类** — 不直接抛出 `HTTPException`
4. **所有配置从环境变量读取** — 启动时验证，快速失败

#### 日志规范

```python
from config.logging_config import get_logger
logger = get_logger(__name__)

# ✅ 正确 — 结构化日志
logger.info("试卷创建成功 | paper_id=%d title=%s", paper.paper_id, paper.title)
logger.error("OCR 识别失败 | student_id=%s error=%s", student_id, str(e), exc_info=True)

# ❌ 错误 — 无结构，无法检索
print(f"试卷创建成功: {paper.title}")

# ❌ 错误 — 永远不要记录密码、Token 等敏感信息
logger.info("用户登录: password=%s", password)
```

#### 数据库操作规范

```python
# ✅ 正确 — 使用依赖注入，事务自动管理
@router.post("/papers")
def create_paper(data: PaperCreate, db: Session = Depends(get_db)):
    paper = Paper(**data.model_dump())
    db.add(paper)
    db.flush()  # 获取 ID，但不提交（由 get_db() 统一提交）
    return {"paper_id": paper.paper_id}

# ❌ 错误 — 手动提交事务
db.add(paper)
db.commit()  # 破坏事务一致性

# ❌ 错误 — N+1 查询
scores = db.query(Score).all()
for s in scores:
    paper = db.query(Paper).filter(Paper.paper_id == s.paper_id).first()

# ✅ 正确 — JOIN 一次性加载
scores = db.query(Score, Paper).join(Paper).all()
```

#### 配置管理规范

```python
# ✅ 正确 — 统一从 settings 对象读取
from config import settings
secret = settings.jwt_secret_key

# ❌ 错误 — 散落的环境变量读取
secret = os.environ.get("JWT_SECRET_KEY", "fallback")
```

### 3.2 前端开发规范

#### 组件三态处理（必须实现）

每个数据驱动组件必须处理三种状态：

```vue
<template>
  <!-- 加载中 -->
  <div v-if="loading" class="loading-skeleton">
    <SkeletonCard v-for="i in 3" :key="i" />
  </div>

  <!-- 错误 -->
  <div v-else-if="error" class="error-state">
    <p>{{ error.message }}</p>
    <button @click="retry">重试</button>
  </div>

  <!-- 空数据 -->
  <div v-else-if="data.length === 0" class="empty-state">
    <p>暂无数据</p>
  </div>

  <!-- 正常渲染 -->
  <div v-else class="data-list">
    ...
  </div>
</template>
```

#### 错误处理规范

```javascript
// ❌ 错误 — alert() 弹窗
catch (e) { alert('操作失败') }

// ❌ 错误 — 静默吞掉错误
catch (e) { data.value = [] }

// ✅ 正确 — 内联错误提示
catch (e) {
  errorMsg.value = e.response?.data?.message || '操作失败，请重试'
}
```

#### API 调用规范

```javascript
// ✅ 正确 — 使用封装的 API 客户端
import { paperApi } from '@/api'
const papers = await paperApi.getPapers()

// ❌ 错误 — 直接使用 axios，缺少拦截器
const res = await axios.get('http://localhost:8000/api/papers')

// ❌ 错误 — 硬编码 URL
const base = 'http://192.168.1.100:8000'
```

#### 状态管理规范

```javascript
// 建议：引入 Pinia（替代目前的 localStorage 散落读取）

// stores/user.js
export const useUserStore = defineStore('user', () => {
  const user = ref(null)
  const token = ref(null)

  function login(userData, accessToken) {
    user.value = userData
    token.value = accessToken
    localStorage.setItem('token', accessToken)
  }

  function logout() {
    user.value = null
    token.value = null
    localStorage.removeItem('token')
  }

  return { user, token, login, logout }
})
```

### 3.3 数据库规范

#### 索引设计原则

1. 高频查询条件字段必须建索引
2. `WHERE` / `JOIN` / `ORDER BY` / `GROUP BY` 中出现的字段优先索引
3. 复合索引遵循"最左前缀"原则
4. 唯一约束自动创建索引（不需要重复创建）

#### 迁移规范

```bash
# 推荐使用 Alembic（替代目前的手动 SQL）
pip install alembic
alembic init migrations
alembic revision --autogenerate -m "add status index to score"
alembic upgrade head
```
