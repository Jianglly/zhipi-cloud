## 一、试卷管理增强（🔴 核心功能缺失）

### ✅ 1.1 试卷创建 API
- `POST /api/papers/` — 教师创建试卷
- 状态：已完成

### 🟡 1.2 试卷编辑 API
```
PUT /api/papers/{paper_id}
```
- 教师修改已创建的试卷信息（标题/科目/总分/日期/描述）
- 已发布的试卷不允许修改（保护数据一致性）
- 涉及文件：`controllers/paper_controller.py`

### 🟡 1.3 试卷删除 API
```
DELETE /api/papers/{paper_id}
```
- 教师删除试卷（软删除 or 仅限 status=0 草稿状态）
- 已关联成绩的试卷不允许删除
- 涉及文件：`controllers/paper_controller.py`

### 🔴 1.4 标准答案录入 API（关键缺失）
```
POST /api/papers/{paper_id}/answer-key
```
- 教师为试卷录入客观题标准答案
- 格式：`{"answers": {"1": "A", "2": "D", "3": "C", ...}}`
- 存入 paper 表的 answer_key 字段（JSON 类型）
- 这是自动批改的**前置依赖**，没有它 OCR 无法对答案
- 涉及文件：`controllers/paper_controller.py`

### 🔴 1.5 试卷详情 API
```
GET /api/papers/{paper_id}
```
- 返回试卷完整信息，**包含 answer_key**
- 目前只有列表接口，前端 MarkingView 需要获取标准答案
- 涉及文件：`controllers/paper_controller.py`

## 二、成绩批改流程增强（🔴 核心功能）

### ✅ 2.1 待批阅列表
- `GET /api/papers/pending` — 返回待批阅列表
- 状态：已完成

### ✅ 2.2 成绩提交
- `POST /api/papers/{paper_id}/submit-score` — 教师录入/审核成绩
- 状态：已完成

### 🟡 2.3 成绩批量提交
```
POST /api/papers/{paper_id}/submit-scores-batch
```
- 教师一次性提交多个学生的成绩
- 入参：`[{"student_id": "xxx", "score": 85, "comment": "..."}, ...]`
- 批阅完一整个班后不需要一个一个点
- 涉及文件：`controllers/paper_controller.py`

### 🟡 2.4 评语保存（MarkingView 评语未入库）
```
修改 submit-score 接口，增加 comment 字段
```
- 当前 MarkingView.vue 里教师写了评语，但提交时不带 comment
- 需要在 score 表增加 `comment` 字段 or 利用 ai_result JSON 字段暂存

### 🟢 2.5 逐题得分录入
```
POST /api/ocr/grade-by-question
```
- 支持人工逐题打分（目前只有总分）
- 格式：`{"question_scores": {"1": 5, "2": 0, "3": 5, ...}}`

## 三、用户管理模块（🟡 增强功能）

### 🟡 3.1 修改密码 API
```
PUT /api/auth/change-password
```
- 入参：old_password + new_password
- 验证旧密码 → bcrypt 加密新密码 → 更新数据库
- 涉及文件：`controllers/auth_controller.py`

### 🟢 3.2 教师查看学生详情
```
GET /api/stats/teacher/student/{student_id}
```
- 教师查看某学生所有历史成绩和趋势
- 涉及文件：`controllers/stats_controller.py`

## 五、安全增强（🟡 生产必备）

### 🔴 5.1 JWT 密钥配置提醒
```
当前 JWT_SECRET_KEY = "dev_secret_key_change_in_production"
```
- 生成一个真正随机的密钥
- 可以通过命令生成：`python -c "import secrets; print(secrets.token_hex(32))"`
- 涉及文件：`.env`

### 🟡 5.2 请求频率限制
```
对登录/OCR 接口加限流
```
- 登录：同一 IP 1分钟内最多5次
- OCR：同一用户1分钟内最多10次（百度 API 有 QPS 限制）
- 涉及文件：`main.py`（使用 slowapi 或手动实现）

### 🟡 5.3 Token 刷新机制
```
POST /api/auth/refresh
```
- 用 refresh_token 换新的 access_token
- 避免用户频繁重新登录
