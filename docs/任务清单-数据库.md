## 一、表结构优化（🟡 完善）

### ✅ 1.1 建表脚本
- `01_create_tables.sql` — 6 张表完整定义
- 状态：已完成

### 🟡 1.2 answer_key 字段类型对齐
```
paper 表的 answer_key 字段类型不一致
```
- SQL 中定义为 `TEXT`
- SQLAlchemy 模型中定义为 `JSON`
- 应统一为 `JSON` 类型（MySQL 5.7+ 原生支持）
- 涉及文件：`01_create_tables.sql` + `models/paper.py`
- 影响：存储客观题答案 `{"1": "A", "2": "D", ...}`，JSON 类型更合适

### 🟡 1.3 评语字段
```
score 表增加 comment 字段
```
- 类型：`VARCHAR(500)`
- 教师批改时写的评语，目前前端写了但无法入库
- 影响：`01_create_tables.sql` + `models/score.py`

### 🟢 1.4 逐题得分字段
```
score 表增加 question_scores 字段
```
- 类型：`JSON`
- 存储每道题的得分：`{"1": 5, "2": 3, "3": 5, ...}`
- 为未来的逐题分析做准备
- 目前可以先不加，等需求明确再做

### 🟢 1.5 软删除支持
```
paper 表和 score 表增加 is_deleted 字段
```
- 类型：`TINYINT(1) DEFAULT 0`
- 删除试卷/成绩时不物理删除，标记软删除
- 保护数据安全

## 二、索引优化（🟡 性能）

### ✅ 2.1 现有索引
- 12 个索引，含唯一索引 + 复合索引 + 覆盖索引
- 状态：已完成

### 🟡 2.2 索引验证
```
确认以下高频查询用到了索引
```
- 学生查自己成绩：`WHERE student_id = ?` → `idx_student`
- 教师查班级成绩：`WHERE class_id = ? AND subject = ?` → `idx_class_subject_score`（覆盖索引，避免回表）
- 按时间查成绩：`WHERE exam_date BETWEEN ? AND ?` → `idx_exam_date_subject`
- 建议：用 `EXPLAIN` 验证实际查询是否走了索引

## 五、数据安全与备份（🟡 运维）

### 🔴 5.1 数据持久化
```
当前数据库数据无法真正更新
```
- 原因：`launcher.py` 和 `init-db.bat` 每次启动检查数据，但 MySQL 是外部服务
- 实际数据存储在 MySQL 中，不会因重启丢失
- 需要和队友**说清楚**：数据在 MySQL 里，不在项目文件里
- 涉及文件：`协作开发指南.md`

### 🟡 5.2 备份脚本
```
新增数据库备份脚本
```
- 一键导出 SQL：`mysqldump zhipi_cloud > backup_YYYYMMDD.sql`
- 一键恢复：`mysql zhipi_cloud < backup_YYYYMMDD.sql`
- 放在 `zhipi-database/backup/` 目录
- 涉及文件：新增 `backup.bat`

### 🟡 5.3 数据迁移脚本
```
数据库结构变更时的迁移方案
```
- 用版本号管理：`V1__init.sql`、`V2__add_comment.sql`、...
- 执行过的迁移不重复执行
- 涉及文件：新增 `migrations/` 目录
